## 1. 需要的指标计算类别汇总

### 1.1 A 股票类指标

| 指标类别 | 具体指标 | 数据需求 | 优先级 |
|---------|---------|---------|------|
| **价格** | 当日涨跌幅、N日收益 | last, prev_close, 历史close | ✓ **能提供** |
| **技术线** | MA5/20/60, 斜率 | 历史close序列 | ✓ **能提供** |
| **动量** | RSI(14), MACD(12,26,9) | 历史close序列 | ✓ **能提供** |
| **波动率** | HV(20), ATR(14), atr_pct | 历史high/low/close | ✓ **能提供** |

| **财务基本面** | PE/PB, ROE, 营收/利润增速 | 财务摘要 | ✓ **能提供** |
| **风险事件** | earnings_within_days, unlock | 日历数据 | ❌ **缺失** |
| **Beta** | beta_60d | 与基准指数的相关性 | ❌ **缺失** |

### 2.2 部分支持 ⚠️ (需要计算/聚合)

| 指标 | 现状 | 改进方案 |

| **隐含波动率** | IV，IV rank, IV percentile | **高** | 从期权价格反解 IV |
| **基金持仓** | 基金前10大持仓 | **中** | 集成 AKShare 或 TuShare 基金持仓 API |
| **ETF 份额流** | 日度 ETF 净申赎 | **低** | 某些成熟 ETF 可查，大部分无 |

| **宏观日历** | 统计局发布日期 | **中** | 维护固定日历或集成 API |
| **持仓主题** | 自定义主题权重 | **低** | 需要 LLM 配置或人工标注 |

### 3.2 中优先级缺口（增强分析质量）

#### 5. 基金持仓和风格暴露

**缺口**: 无基金前 10 大持仓、行业暴露、持仓集中度  
**影响**: `get_asset_snapshots` 基金段的信息不完整  
**建议方案**: 集成 TuShare 或 Wind API 的基金持仓

---

#### 6. Beta 值计算

**缺口**: 无与基准指数的相关性数据  
**影响**: 无法评估股票系统风险  
**建议方案**:
```python
def calculate_beta(stock_returns, benchmark_returns, period=60):
    """计算 60 日 beta"""
    import numpy as np
    return np.cov(stock_returns, benchmark_returns)[0][1] / np.var(benchmark_returns)
```

**成本**: ⭐ 低（numpy 即可）

---

#### 7. 利率曲线和 SHIBOR

**缺口**: 无国债收益率、货币市场利率  
**影响**: `get_market_context` 中 liquidity_regime 无法精确判断  
**建议方案**: 集成央行或交易所数据源
